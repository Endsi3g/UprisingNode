Introduction

Bienvenue sur l'API Uprising Node

L'API Uprising Node est conçue autour des principes REST. Notre API utilise des codes de réponse HTTP standard, supporte le partage de ressources cross-origin (CORS) et renvoie des données structurées en JSON. C'est le socle technique pour une intégration transparente avec notre infrastructure bancaire B2B.

info

Toutes les requêtes API doivent être effectuées via HTTPS. Les appels via HTTP simple échoueront.

Authentification

L'authentification s'effectue via des clés API Bearer. Vous pouvez gérer vos clés dans le tableau de bord développeur. Ne partagez jamais vos clés secrètes.

Requête

cURL 
curl https://api.uprisingnode.com/v1/comptes \
  -H "Authorization: Bearer sk_live_..." \
  -H "Content-Type: application/json"

Créer un virement

Cet endpoint initie une demande de virement vers un compte externe pré-approuvé. La réponse est synchrone mais le traitement est asynchrone.

Post /v1/virements

Paramètres

montant Entier Requis

Montant à transférer en centimes. Par exemple, pour 10,00 $, envoyez 1000.

devise String Requis

Code ISO à 3 lettres de la devise. Actuellement supporte usd, eur.

destination String

ID du compte de destination. Si omis, utilise le compte par défaut.

Réponse

JSON 
{
  "id": "tr_1M5k9A2eZvKYlo2CLp",
  "object": "transfer",
  "amount": 425000,
  "currency": "usd",
  "status": "pending",
  "created": 1678892445,
  "destination": {
    "id": "ba_1M5k8z2eZvKYlo2C",
    "bank_name": "Banque Transatlantique",
    "last4": "4242"
  }
}

Codes d'Erreur

Uprising Node utilise les codes de réponse HTTP conventionnels pour indiquer le succès ou l'échec d'une requête API.

Code Description
200 OK - Tout a fonctionné comme prévu.
400 Bad Request - La requête était inacceptable (souvent un paramètre manquant).
401 Unauthorized - Aucune clé API valide fournie.
402 Request Failed - Les paramètres étaient valides mais la requête a échoué.
404 Not Found - La ressource demandée n'existe pas.
